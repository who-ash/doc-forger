# Documentation Manager

This folder tracks implementation history in a way that is easy to revisit later.

## Structure
- `commands/`: chronological command logs for major setup and change sessions.
- `lessons-learned.md`: mistakes, corrections, and prevention notes.
- `cases/`: grouped docs by product area:
  - `admin/`
  - `user/`
  - `organization/`
- `.cursor/skills/`: reusable task skills for implementation quality.
- `.cursor/rules/`: always-on governance and documentation rules.

## Usage
After meaningful changes:
1. Add/update a command log file.
2. Update lessons learned if any issue or correction happened.
3. Update relevant case files with current behavior and next steps.
